Abstract

This project presents a simple yet effective web-based system that allows users to upload medical prescriptions and extract medicine names using Optical Character Recognition (OCR). The system then analyzes these names to provide basic information such as uses, dosage, and warnings. Built using HTML, CSS, JavaScript, it demonstrates how digital tools can simplify healthcare interactions for patients.

Introduction

In today's digital healthcare landscape, patients often receive prescriptions that are either difficult to read or not well understood. This project addresses this gap by enabling users to upload images of prescriptions and get relevant medicine details instantly. The application is designed with accessibility and simplicity in mind, ensuring anyone with a basic device and internet access can use it.

Problem Statement

To develop a web-based tool where users can upload a prescription image, extract the text using OCR, identify the medicine names mentioned, and display their related information such as usage, dosage, and warnings. The system aims to improve health literacy and ensure patients understand their treatment.

Objectives

  • Provide a simple UI to upload prescriptions in image format.
  • Use OCR to extract text from images using Tesseract.js.
  • Match extracted text with a predefined medicine database.
  • Display detailed medicine information clearly.

Technologies Used

HTML
CSS
JavaScript
Tesseract.js
  • HTML – for structure of the webpage
  • CSS – for styling and layout
  • JavaScript – for interactivity and logic
  • Tesseract.js – to implement OCR in browser

System Overview

The application comprises a frontend where the user can upload an image file. JavaScript captures the file and uses the Tesseract.js library to perform OCR on the image. Extracted text is compared against a list of common medicines stored in the script. If matches are found, a result card is generated showing the medicine's name, uses, dosage, and warnings. Otherwise, an error message is shown.

Key Features

  • Upload Prescription Button – Users can upload image files through a clean UI.
  • Text Extraction – Tesseract.js scans the prescription for text content.
  • Medicine Detection – Matches words from the text with known medicines.
  • Result Display – Detailed medicine cards with name, uses, dosage, and warnings.
  • Responsive Layout – Visually styled using CSS for user-friendly experience.

Challenges Faced

  • Handling OCR inaccuracies due to poor image quality or handwriting.
  • Ensuring responsive design across various screen sizes.
  • Mapping only relevant medicine names while filtering noise in OCR results.

Learning Outcomes

This project helped strengthen knowledge in core frontend development, understanding how OCR works with JavaScript, and designing real-world healthcare utility tools. It also highlighted the importance of accessibility and user experience design.

Conclusion

The digital prescription scanner offers a convenient way for users to interpret handwritten or printed prescriptions. While currently limited to a fixed set of medicines, it demonstrates the potential for building intuitive, browser-based healthcare tools that can educate and empower users about the medication they are prescribed.

Future Scope

  • Expand medicine database to cover more drugs.
  • Enable OCR tuning for different languages and fonts.
  • Store history of scanned results securely for user review.
  • Improve matching accuracy with fuzzy search or NLP methods.
  • Provide the location of the nearest medicine shop.
